# Fixture: SaaS Ops Dashboard Brief

Build a first-screen operational dashboard for a support and billing team.

The page should help a user answer:

- What needs attention now?
- Which accounts are at risk?
- What changed since the last review?

## Product Shape

- Dense product UI, not a landing page
- Quiet, utilitarian visual style
- Real table, status blocks, and summary metrics
- Clear primary action and clear secondary actions
- Compact but readable on desktop and mobile

## Required Content

- One summary strip with 3 to 4 key metrics
- One prioritized action list
- One table or grid of accounts
- One small chart or trend block
- One state area for empty, loading, or error messaging

## Visual Rules

- No blue or purple gradient abuse
- No giant hero section
- No card-inside-card stacking
- No oversized corner radius
- No decorative imagery that does not explain the data
- No monochrome wall with weak hierarchy
- No dashboard layout that becomes airy and empty

## Interaction Rules

- Hover and focus states for all actions
- Active or pressed state for all primary actions
- Disabled state for actions that are temporarily unavailable
- Visible focus appearance and 24 x 24 px minimum pointer targets
- Row or item highlight on interaction
- Empty state for no urgent items
- Loading state for delayed data
- Error state for failed account fetch

## Review Targets

When Codex uses the boost pack correctly, the result should show:

- A clear page type
- Strong information hierarchy
- Good density for an operational task
- No overflow on desktop or mobile
- No missing states
- A visual style that feels like a working tool, not a promo page
